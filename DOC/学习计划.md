# simpleFFplay 音视频学习计划

> 目标：通过抄写 simpleFFplay 项目，掌握音视频开发核心技术
> 节奏：每天约 300 行，共 2148 行，约 8 天完成
> 原则：头文件和对应实现配对抄写，每天完成一个完整功能单元

---

## 项目整体架构

```
容器格式文件 (MP4/MKV/AVI...)
        │
        ▼
  demux_thread          ← demux.c：读文件，识别流，分发 packet
        │
   ┌────┴────┐
   ▼         ▼
audio_pkt  video_pkt    ← packet.c：线程安全的编码数据队列
   │         │
   ▼         ▼
audio_dec  video_dec    ← audio.c / video.c：解码线程
   │         │
   ▼         ▼
audio_frm  video_frm    ← frame.c：线程安全的解码帧队列
   │         │
   ▼         ▼
SDL音频回调  video_playing_thread
                │
                ▼
           SDL2 渲染到屏幕
                ↑
        player.c：音视频同步、事件控制
```

**4 个并发线程：**

| 线程 | 位置 | 职责 |
|------|------|------|
| demux_thread | demux.c | 读文件，分发 packet |
| audio_decode_thread | audio.c | 解码音频 → frame 队列 |
| video_decode_thread | video.c | 解码视频 → frame 队列 |
| video_playing_thread | video.c | 按时机渲染视频帧 |
| SDL 音频回调 | audio.c | SDL 驱动，填充音频设备 |

## Day 1 — 线程安全队列（~297 行）

### 抄写文件

| 文件 | 行数 | 职责 |
|------|------|------|
| packet.h | 14 | 编码数据队列接口 |
| packet.c | 139 | 链表队列实现 |
| frame.h | 20 | 解码帧队列接口 |
| frame.c | 124 | 环形缓冲区实现 |

### 为什么从这里开始

整个播放器有 4 个线程并发运行，线程之间只通过这两个队列通信。
先搞懂队列，后面的 demux、audio、video 才有意义。

### 学习重点

**1. 两种队列的区别**

```
packet_queue（链表）         frame_queue（环形缓冲）
─────────────────            ──────────────────────
存编码后的原始数据            存解码后的 AVFrame
大小不固定，动态分配           大小固定（3帧视频/9帧音频）
生产快消费慢时可以堆积          满了就阻塞生产者
```

**2. 线程安全：SDL_mutex + SDL_cond**

```c
// 生产者（demux_thread）放入数据
SDL_LockMutex(q->mutex);
// ... 操作链表 ...
SDL_UnlockMutex(q->mutex);
SDL_CondSignal(q->cond);      // 唤醒等待的消费者

// 消费者（decode_thread）取数据
SDL_LockMutex(q->mutex);
while (queue_empty && !abort) {
    SDL_CondWaitTimeout(q->cond, q->mutex, 20); // 最多等 20ms
}
// ... 取数据 ...
SDL_UnlockMutex(q->mutex);
```

**3. flush packet 的作用**

Seek 时需要丢弃队列里的旧数据，并通知解码器重置状态：
```c
packet_queue_flush();           // 清空队列中的旧 packet
packet_queue_put_nullpacket();  // 注入特殊标记，解码器收到后调用
                                // avcodec_flush_buffers() 重置自身
```

**4. frame_queue 的 keep_last 设计**

暂停时，读指针不前进，最后一帧保持在屏幕上持续显示。

**5. AVFrame 生命周期**

```c
av_frame_alloc()    // 分配
av_frame_unref()    // 释放内部数据（可复用结构体）
av_frame_free()     // 释放结构体本身
```
frame.c 在 `frame_queue_next()` 中封装了 unref，使用者不需要手动管理。

| demux.c | 229 | 读文件、识别流、分发 packet |

### 学习重点

**1. 容器格式 vs 编码格式**

```
文件（MP4）= 容器
    ├── 视频轨 = H.264 / H.265 编码的 packet
    └── 音频轨 = AAC / MP3 编码的 packet

libavformat：负责拆容器，输出 packet（还是编码的）
libavcodec：负责解码 packet，输出原始帧（YUV / PCM）
```

**2. 打开文件流程**

```c
avformat_open_input()         // 打开文件，识别容器格式
avformat_find_stream_info()   // 分析流信息（可能需要读几帧才能确定）
av_find_best_stream()         // 找最佳音频流和视频流，返回 stream_index
```

**3. demux_thread 主循环**

```c
while (1) {
    // 背压控制：队列太满就暂停读取
    if (stream_has_enough_packets()) { SDL_Delay(10); continue; }

    // 处理 Seek 请求
    if (is->seek_req) {
        av_seek_frame(...);
        packet_queue_flush(&audio_queue);
        packet_queue_flush(&video_queue);
        packet_queue_put_nullpacket(...); // 通知解码器重置
    }

    // 读一个 packet，按 stream_index 分发
    av_read_frame(fmt_ctx, pkt);
    if (pkt->stream_index == audio_stream_index)
        packet_queue_put(&audio_queue, pkt);
    else if (pkt->stream_index == video_stream_index)
        packet_queue_put(&video_queue, pkt);
    else
        av_packet_unref(pkt); // 字幕流等其他轨道，丢弃
}
```

**4. 中断回调**

```c
// 注册 interrupt_callback，让 av_read_frame 可以被中断退出
// 否则网络流卡住时，程序无法响应退出指令
fmt_ctx->interrupt_callback.callback = decode_interrupt_cb;
fmt_ctx->interrupt_callback.opaque = is;
```

**1. 打开解码器**

```c

avcodec_parameters_to_context()     // 把流参数拷进上下文
```

**2. 新版解码 API（send / receive 模式）**

```c
// 旧 API（已废弃）：avcodec_decode_audio4()
// 新 API：
avcodec_send_packet(codec_ctx, pkt);      // 送入一个 packet
avcodec_receive_frame(codec_ctx, frame);  // 取出一个解码帧

// 一个 packet 可能对应多个 frame，需要循环 receive：
while (avcodec_receive_frame(...) == 0) {
    // 处理这一帧
}
```

**3. 音频参数**

```
采样率 sample_rate：每秒采样次数，如 44100 Hz
位深   format：     每个采样点的精度，如 S16（16位有符号整数）
声道数 channels：   单声道=1，立体声=2
帧大小 nb_samples： 一帧包含的采样点数
```

**4. 平面格式 vs 交错格式（重要！）**

```
交错格式 (packed)：L R L R L R ...   ← SDL 需要此格式
平面格式 (planar)：L L L ... R R R ...

AV_SAMPLE_FMT_S16   = 交错 16bit（SDL 直接兼容）
AV_SAMPLE_FMT_S16P  = 平面 16bit（需要 swresample 转换）
AV_SAMPLE_FMT_FLTP  = 平面 float（AAC 解码默认输出，需要转换）
```

|------|------|------|
| video.h | 9 | 视频模块接口 |
| video.c 前半 | ~270 | 打开解码器 + SDL 窗口 + 解码线程 |

### 抄写范围

`open_video_stream()` + `open_video_playing()` + `video_decode_thread()`

### 学习重点

**1. SDL2 视频渲染三件套**

```
SDL_Window    →  操作系统窗口
SDL_Renderer  →  GPU 渲染接口
SDL_Texture   →  GPU 显存中的图像（YUV 格式）

渲染流程：
AVFrame(YUV420P)
    → SDL_UpdateYUVTexture()   把 YUV 数据上传到 GPU
    → SDL_RenderCopy()         把纹理绘制到渲染器
    → SDL_RenderPresent()      呈现到屏幕（交换缓冲区）
```

**2. YUV 格式**

```
YUV420P（最常见的视频格式）：
    Y：亮度分量，每个像素一个值
    U/V：色度分量，每 4 个像素共享一组

存储量 = W×H × 1.5 字节（比 RGB 的 W×H×3 少一半）
人眼对色度不敏感，所以降采样后画质损失极小

SDL_PIXELFORMAT_IYUV = YUV420P，SDL 原生支持，不需要 CPU 转换
```

**3. 视频帧时间戳**

```c
double pts = frame->pts * av_q2d(stream->time_base);
// av_q2d 把 AVRational（分数）转换成 double
// 例如 time_base = 1/90000，pts = 90000 → 实际时间 = 1.0 秒
```

## Day 6 — 视频播放 + 音视频同步（~270 行，video.c 后半）

### 抄写范围

`video_refresh()` + `compute_target_delay()` + `video_display()`

### 学习重点

**1. 音视频同步的核心思路**

以音频时钟为主（音频跳帧会被人耳明显察觉，视频丢帧不容易被感知）：

```
diff = video_clock - audio_clock

diff < -0.04s  →  视频落后：缩短等待时间，加速追赶
diff > +0.04s  →  视频超前：延长等待时间，等音频跟上
|diff| < 0.04s →  在容忍范围内，不调整
```

**2. compute_target_delay() 完整逻辑**

```c
double compute_target_delay(double delay, player_stat_t *is) {
    double diff = get_clock(&is->video_clk) - get_clock(&is->audio_clk);

    // 同步阈值随帧间隔自适应（帧率越低，容忍越大）
    double sync_threshold = FFMAX(AV_SYNC_THRESHOLD_MIN,
                                  FFMIN(AV_SYNC_THRESHOLD_MAX, delay));

    if (diff <= -sync_threshold)                            // 视频慢了
        delay = FFMAX(0, delay + diff);                     // 缩短等待
    else if (diff >= sync_threshold && delay > FRAMEDUP_THRESHOLD)
        delay = delay * 2;                                  // 视频快了，加倍等待

    return delay;
}
```

**3. video_playing_thread 帧显示决策**

```
每 10ms 触发一次 video_refresh()：

取出下一帧的 pts
    ↓
计算 delay = compute_target_delay(帧间隔)
    ↓
remaining_time = (上次显示时刻 + delay) - 当前时刻
    ↓
remaining_time > 0 → 本轮不显示，继续等待
remaining_time ≤ 0 → 调用 video_display() 渲染到屏幕
                      更新 video_clock
```

## Day 7 — 播放器数据结构 + 时钟/暂停（~300 行）

### 抄写文件

| 文件 | 行数 | 职责 |
|------|------|------|
| player.h | 200 | 所有核心数据结构和常量 |
| player.c 前半 | ~100 | 时钟系统 + 暂停逻辑 |

### 学习重点

**1. player_stat_t 是整个播放器的大脑**

```c
typedef struct player_stat_t {
    // 时钟
    play_clock_t audio_clk;    // 音频时钟（基准）
    play_clock_t video_clk;    // 视频时钟（跟随音频）

    // 队列（各两套：packet + frame）
    packet_queue_t audio_pkt_queue;
    frame_queue_t  audio_frm_queue;
    packet_queue_t video_pkt_queue;
    frame_queue_t  video_frm_queue;

    // 解码器上下文
    AVCodecContext *p_acodec_ctx;
    AVCodecContext *p_vcodec_ctx;

    // 状态标志
    int paused;
    int seek_req;
    int64_t seek_pos;
    int abort_request;
} player_stat_t;
```

**2. 时钟系统设计**

```c
typedef struct {
    double pts;          // 最近一次设置的时间点
    double pts_drift;    // pts 与系统时间的差值
    int serial;          // Seek 后递增，用于丢弃旧数据
} play_clock_t;

// 读时钟：pts_drift + 当前系统时间 = 当前播放时刻
double get_clock(play_clock_t *c) {
    return c->pts_drift + av_gettime_relative() / 1e6;
}

// 写时钟
void set_clock(play_clock_t *c, double pts) {
    c->pts = pts;
    c->pts_drift = pts - av_gettime_relative() / 1e6;
}
```

**3. 暂停的难点：时钟漂移**

```
暂停时：停止更新时钟，记录暂停时刻的 pts

恢复时：必须重新设置 pts_drift
        否则 get_clock() 会把暂停期间流逝的系统时间也算进去
        导致时钟跳跃，视频瞬间追赶，画面跳帧
```

## Day 8 — 事件循环 + Seek（~191 行，player.c 后半）

### 抄写范围

`player_running()` 主循环、事件处理、进度条渲染

### 学习重点

**1. SDL 事件循环**

```c
while (1) {
    SDL_PollEvent(&event);       // 非阻塞，立即返回
    switch (event.type) {
        case SDL_KEYDOWN:
            switch (event.key.keysym.sym) {
                case SDLK_ESCAPE: quit(); break;
                case SDLK_SPACE:  toggle_pause(); break;
                case SDLK_RIGHT:  stream_seek(+10 * AV_TIME_BASE); break;
                case SDLK_LEFT:   stream_seek(-10 * AV_TIME_BASE); break;
            }
        case SDL_WINDOWEVENT: handle_resize(); break;
        case SDL_QUIT: quit(); break;
    }
    // 渲染进度条
    SDL_Delay(100);   // 主线程 100ms 轮询一次，不占 CPU
}
```

**2. Seek 完整流程**

```
用户按方向键
    │
    ▼
stream_seek() 设置 seek_pos、seek_req 标志
    │
    ▼
demux_thread 检测到 seek_req
    │
    ├─ av_seek_frame() 跳转到目标位置（I 帧对齐）
    ├─ packet_queue_flush() 清空两个队列的旧数据
    └─ packet_queue_put_nullpacket() 注入 flush 包
            │
            ▼
    decode_thread 收到 flush 包
            │
            └─ avcodec_flush_buffers() 重置解码器内部状态
                    │
                    ▼
            serial 递增 → frame_queue 中旧帧自动被丢弃
```

**3. 进度条渲染**

```c
// 获取当前播放时刻
double current = get_clock(&is->audio_clk);
// 获取总时长
double duration = fmt_ctx->duration / AV_TIME_BASE;
// 计算比例，用 SDL 画矩形
```
